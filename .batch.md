# Batch: Kill Web UI, Make Bot-Only System

## Task 1: Remove Express server and frontend
Delete all Express server code and frontend code. Remove:
- src/server/server.ts
- src/frontend/ (entire directory)
- src/server/server.test.ts (if exists)
- src/frontend/toolProvider.test.ts

Keep all other server files (db.ts, extraction.ts, briefing.ts, etc.) as they'll be imported by the bot.

Update package.json:
- Remove "dev:server" and "dev:frontend" scripts
- Remove "start" script
- Keep "dev" but change it to just run the bot
- Remove express, cors, and other server-only dependencies from package.json (but keep them in node_modules for now, we'll clean up later)

## Task 2: Refactor bot and implement all commands
Update src/bot/index.ts to import and use server logic directly, then create src/bot/commands.ts with all bot commands.

Bot refactoring:
- Import extraction functions from src/server/extraction.ts
- Import briefing functions from src/server/briefing.ts
- Import DB functions from src/server/db.ts
- Remove any HTTP-related code
- Ensure bot can trigger extraction, get briefing, query activities

Command infrastructure (create src/bot/commands.ts):
- Parse incoming messages for /command pattern
- Route to appropriate handler
- Track conversation state (for multi-step flows like delivery approval)
- Use in-memory Map to track pending approvals: Map<chatId, {activityId, step}>

Commands to implement:
1. /extrair
   - Call extraction logic directly
   - Auto-commit extracted activities as 'pendente' status
   - Show summary: "Encontrei X atividades, Y duplicatas removidas"
   - If no activities found, show suggestive message: "Nenhuma atividade encontrada em N mensagens. Última extração foi há X horas. Executar novamente?"

2. /atividades [status]
   - Query activities from DB
   - Optional status filter: /atividades [pendente|concluido|descartado]
   - Format as WhatsApp message with activity list
   - Show: ID, type, title, due date, status
   - Sort by due date

3. /concluir <id>
   - Update activity status to 'concluido'
   - Auto-approve (no confirmation needed)
   - Show confirmation: "Atividade X marcada como concluída"

4. /descartar <id>
   - Update activity status to 'descartado'
   - Auto-approve (no confirmation needed)
   - Show confirmation: "Atividade X marcada como descartada"

5. /editar <id> <field> <value>
   - Allowed fields: title, due_date, type, confidence
   - Auto-approve (no confirmation needed)
   - Show confirmation with updated activity details

6. /resumo
   - Query activities due in next 7 days
   - Format as WhatsApp message
   - Sort by due date
   - Show: type, title, due date, status
   - If no upcoming activities, show "Nenhuma atividade pendente nos próximos 7 dias"

7. /entregar <id>
   - Generate delivery draft using LLM
   - Save draft to DB immediately (delivery_draft field)
   - Show draft to user
   - Ask "Aprovar? (sim/não/editar)"
   - Handle responses:
     - "sim" → send delivery, mark as concluido
     - "não" → discard, keep draft in DB
     - "editar <text>" → update draft, show again, ask again
   - Command priority: any /command cancels the flow but keeps the draft
   - Use in-memory state to track pending approvals

Integration:
- Update src/bot/index.ts to import and use commands.ts
- Parse incoming messages, check if they're commands or replies to pending approvals
- Route accordingly
