# Batch: Rework to OpenUI (Full Node.js/TypeScript)

Rework the entire project to Node.js/TypeScript with vanilla OpenUI for the interface. Replace Streamlit with OpenUI, port all Python logic to TypeScript, unify bot and app.

Current structure:
- `bot/` - Node.js WhatsApp listener
- `app/` - Streamlit Python UI + Python backend

New structure:
- `bot/` - Merged into unified Node.js app
- `app/` - Node.js/TypeScript OpenUI web app + backend (extraction, prompts, db logic)
- Full stack: vanilla OpenUI frontend, Express backend, TypeScript throughout

## Task 1: Set up unified Node.js/TypeScript project structure
Initialize a monorepo-style project structure: replace bot/ and app/ with a unified Node.js/TypeScript codebase. Set up:
- Root package.json with shared dependencies (TypeScript, Express, SQLite, OpenUI)
- src/bot/ - WhatsApp listener (refactored from bot/index.js)
- src/server/ - Express backend server
- src/frontend/ - OpenUI vanilla JavaScript/HTML frontend
- Vite config for frontend bundling
- tsconfig.json for the whole project
- .env config file

## Task 2: Port database and schema to TypeScript
Convert `app/lib/db.py` to TypeScript (src/server/db.ts). Create:
- SQLite connection with WAL mode
- Message CRUD operations
- Activity CRUD operations
- Database schema setup (from shared/schema.sql)
- Helper functions for fetch_activities, fetch_messages, etc.

## Task 3: Port extraction and prompts to TypeScript
Convert `app/lib/extraction.py` and `app/lib/prompts.py` to TypeScript:
- run_extraction() function with batching logic
- LLM API calls to OpenCode
- Deduplication and validation logic
- Prompt building with timezone handling (São Paulo)
- Token counting and cost calculation

## Task 4: Create Express backend server
Build src/server/server.ts with REST API endpoints:
- GET /api/activities (filters, pagination)
- GET /api/messages (pagination)
- POST /api/extract (trigger extraction, return token usage)
- GET /api/stats (cost, messages remaining, queue status)
- Integrate database and extraction modules
- Add error handling and logging

## Task 5: Build Home/Dashboard OpenUI page
Create src/frontend/pages/dashboard.html with vanilla OpenUI components:
- Activity list (type badges, status, confidence)
- Filter/sort controls
- Cost and token usage stats display
- "Atualizar" button to trigger extraction
- Pagination for activity list
- Wire up API calls to backend

## Task 6: Build Messages OpenUI page
Create src/frontend/pages/messages.html with vanilla OpenUI:
- Message feed with timestamps
- Group labels (alunos/profs)
- Author names and bodies
- Processed/unprocessed indicators
- Pagination
- API integration for message fetching

## Task 7: Build Status OpenUI page
Create src/frontend/pages/status.html showing:
- System statistics (total messages, activities, extraction progress)
- Token usage and estimated costs
- Last extraction timestamp
- Queue status
- Simple, read-only display

## Task 8: Refactor WhatsApp bot to TypeScript
Convert bot/index.js to src/bot/index.ts:
- Use existing whatsapp-web.js library
- Connect to same SQLite database
- Keep group IDs from .env
- Maintain QR code login flow
- Database connection with proper timeouts

## Task 9: Implement frontend theming and styling
Apply OpenUI's default theme to all pages. Create src/frontend/styles/ with:
- Main CSS file with OpenUI defaults
- Layout grid/flexbox utilities
- Consistent spacing and typography
- Dark/light mode support if OpenUI provides it

## Task 10: Set up Docker and docker-compose
Update Dockerfile and docker-compose.yml to:
- Install Node.js 18+ and build TypeScript
- Run unified server (WhatsApp bot + Express server together)
- Build OpenUI frontend
- Share SQLite volume
- Expose ports for web UI
- Mount .env file for configuration

## Task 11: Integration testing and cleanup
Test the complete flow:
- WhatsApp bot receives messages and writes to DB
- Extract button works and updates activities
- All pages load and display data correctly
- Filters and pagination work
- Delete old Python code (app/lib, app/pages, app/Home.py, etc.)
- Verify no Streamlit dependencies remain
